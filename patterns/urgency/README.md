# Urgency Patterns

5-8 patterns that frame time, scarcity, or activity to motivate immediate action. Use these carefully; manipulative urgency damages trust.

## Patterns

| # | Name | Subcategory |
|---|---|---|
| 01 | Countdown timer (deadline-based) | Time |
| 02 | Stock scarcity indicator | Quantity |
| 03 | Activity-based urgency (X people viewing) | Activity |
| 04 | Limited-time offer banner | Time |
| 05 | Early access window | Time |
| 06 | Waitlist position display | Quantity |
| 07 | Real-time recent activity ticker | Activity |
| 08 | Cohort enrollment deadline | Time |

## Anti-patterns documented in this category

Several urgency patterns commonly used in low-trust e-commerce are documented as ANTI-PATTERNS to avoid:

- Fake countdown timers that reset on page reload
- Stock scarcity claims that are always shown
- "Other people are looking at this" notifications that are fabricated

These are documented to help builders avoid them, not as recommendations.

## How to use this category

Honest urgency works (real deadlines, real scarcity, real activity). Dishonest urgency converts short-term and burns trust long-term. The threshold question: would you be comfortable explaining the urgency claim to a user who asked how it's calculated? If not, don't use it.
