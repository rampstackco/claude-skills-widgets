# Lead Capture Patterns

12-15 patterns for capturing user information across forms, modals, gates, and progressive disclosure mechanisms.

## Patterns

| # | Name | Subcategory |
|---|---|---|
| 01 | Inline single-field form | Form |
| 02 | Inline multi-field form | Form |
| 03 | Multi-step progressive form | Form |
| 04 | Modal lead form | Modal |
| 05 | Exit-intent modal form | Modal + trigger |
| 06 | Content gate (download in exchange for email) | Gate |
| 07 | Progressive content gate (partial visibility) | Gate |
| 08 | Newsletter signup inline | Newsletter |
| 09 | Newsletter signup footer | Newsletter |
| 10 | Sticky bottom newsletter bar | Newsletter |
| 11 | Social login (Google, GitHub, etc.) | Auth-based |
| 12 | Magic link signup | Auth-based |
| 13 | Calendar booking lead capture | Booking |
| 14 | Chat-based lead capture | Conversational |
| 15 | Free trial signup with no credit card | Trial |

## How to use this category

Define what you actually need from the user (email, name, role, company, all of these). Match the friction to the value: ungated content needs no form; high-value lead magnets justify multi-field forms.
