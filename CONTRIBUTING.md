# Contributing

Pattern submissions and corrections welcome. This repo follows a research-backed standard: every pattern must be grounded in real-world examples with attribution, not invented or speculated.

## What we accept

- New patterns within existing categories (cta, lead-capture, social-proof, urgency, interactive-tooling)
- New variations of existing patterns
- Corrections to attribution, examples, or implementation notes
- Updated real-world examples when brands redesign or remove a pattern
- New category proposals (open an issue first to discuss scope)

## What we don't accept

- Patterns without at least 3 real-world examples with attribution
- Patterns ungrounded in current usage (purely historical patterns belong in a separate archive)
- Branded or promotional examples (no self-promotion via pattern submission)
- Code implementations (this catalog is documentation only; component implementations belong in a separate companion repo planned for future work)

## Submission format

Patterns are markdown files following the schema in `patterns/SCHEMA.md`. Use one of the existing pattern files in `patterns/cta/`, `patterns/lead-capture/`, etc. as a template.

Each submission includes:

1. Pattern file in the appropriate category folder
2. Update to the category README listing the new pattern
3. PR description noting the 3+ real-world examples and the source for each

## Quality standards

- Voice: direct, declarative, no marketing fluff
- Length: 200-500 words per pattern file
- Examples: 3 minimum, 5-7 ideal
- Variations: 2-4 named sub-patterns when meaningful
- Cross-references: link to related patterns and to relevant claude-skills skills
